# Reader: adr-history

## Summary

Koi's decision record is a 13-ADR corpus plus a plans/proposals trail that tells a clear two-phase story. Phase one is reconstructed history: ADRs 001-010 were all committed in a single batch on 2026-02-15 (commit 8a63730), eight days after the repo's first commit (2026-02-07), with backdated decision dates at suspiciously regular intervals (2025-01-15, 2025-06-01, 2025-09-01... 2025-12-15). They are decision archaeology written after the fact, not contemporaneous records. Phase two is real-time: ADR-011 (security overhaul, 2026-03-18), ADR-012 (consolidation roadmap, 2026-03-25), and ADR-013 (runtime adapters, 2026-03-26) each landed with the work they describe. The whole project — 15 crates, ~70 endpoints, ~50 CLI commands, 796 test annotations — was built in roughly 3 months and 176 commits via an explicit human+AI pipeline (proposals signed "Leo Botinelly, Claude" and "operator + AI"; ADR-011 derives from a "6-agent codebase review").

The decision process is unusually self-critical and, crucially, its claims mostly verify against the code. ADR-011 documents 87 findings including an RCE vector (set-hook executed via sh -c), a cryptographically broken FIDO2 unlock slot, and an empty-passphrase CA promotion protocol — and then nearly all of it was executed within one week (security commit storm 03-18 to 03-25). I verified: exactly 1 production-path unwrap/expect remains across all 15 crates (crates/koi/src/commands/certmesh.rs:1206); dat_auth_middleware exists (crates/koi/src/adapters/http.rs:458); mtls.rs exists; FIDO2 is feature-gated off (crates/koi-crypto/Cargo.toml:47); cross-domain Cargo deps are gone. However, the record contains one significant confabulation: ADR-012 Block 4 says cross-domain imports never existed ("implemented correctly from the start... ADR-011's description was prospective"). Git proves otherwise — at ADR-011's commit (588b616), koi-health depended on 4 domain crates, and commit aa979d4 (same day) removed 8 cross-domain deps. The consolidation auditor lost track of its own week-old fix and rewrote history. This is the signature failure mode of AI-maintained documentation: high fidelity to current code, low fidelity to causal history.

The ADR-010 → ADR-013 reversal (Docker adapter rejected, then runtime adapters built in-core) is reasonably argued: ADR-010 rejected a sidecar; ADR-013 reverses because the embedded facade (ADR-008) and single-binary architecture made in-process integration the natural shape, and a sidecar would be invisible to embedded consumers like zen-garden. The reversal honestly lists costs, but ADR-010's central security objection — Docker socket grants root-equivalent access — is waved through as "documented, not mitigated" (ADR-013 line 124). ADR-013 also over-promises: its metadata table maps K8s/systemd/Incus label namespaces, but only DockerBackend exists; Systemd/Incus/Kubernetes are stub enum variants returning BackendUnavailable (crates/koi-runtime/src/lib.rs:252-258). The orchestration itself is real and correctly placed in the binary crate (crates/koi/src/orchestrator.rs, 458 lines).

ADR-012 is effectively complete: Blocks 1-3, 5 done with verifiable commits (11b8311, 97f64e8, d223a35, dc843b6); Blocks 4/6 skipped (6's "already implemented" claims all verify); Block 7 partially done (factory-reset, SelectMany, vault auto-unlock) with FIDO2 redesign, SSE replay, and dashboard enhancements deferred. Load-bearing decisions: ADR-001 (host-as-mDNS-bridge, the founding premise), ADR-002 (verb-oriented JSON), ADR-005 (lease lifecycle), ADR-003 (envelope encryption), ADR-008 (embedded facade — it enabled both the zen-garden consumer and the in-process test strategy that replaced fragile shell scripts). Speculative/over-built: ADR-007's command-surface crate (its channel-binding registry and parity validation were superseded or never built — COMMAND-MANIFEST phases 5-9 marked superseded/deferred), the FIDO2 half of ADR-009 (shipped, then declared broken by ADR-011, now dormant behind a feature flag), and the non-Docker runtime backends. The only proposal never implemented in any form is DOCKER-ADAPTER.md (status "Idea", absorbed by ADR-013 but never marked superseded).

## Strengths

- ADR claims verify against code at an unusually high rate: panic elimination is real (exactly 1 production-path unwrap left across 15 crates, at crates/koi/src/commands/certmesh.rs:1206), DAT middleware exists (crates/koi/src/adapters/http.rs:458), mtls.rs exists, FIDO2 is gated (crates/koi-crypto/Cargo.toml:47), cross-domain Cargo deps are zero today
- ADR-011 is a rare act of architectural self-indictment: it documents its own RCE vector (set-hook via sh -c), broken FIDO2 crypto, and empty-passphrase promotion in plain language, and ~90% of it was executed within one week (commits 588b616 through dc843b6, 2026-03-18 to 03-25)
- Negative-consequences sections are substantive, not perfunctory: ADR-013 admits 'Docker socket access implies root-equivalent permissions — documented, not mitigated'; ADR-012 admits 'Block 4 motivation must be architectural clarity, not feature delivery' and that FIDO2 is 'an unsolved problem across the Rust ecosystem'
- The ADR-010 → ADR-013 reversal is explicit and traceable: ADR-013 names what changed (sidecar assumption invalidated by embedded facade + single-binary strength), keeps ADR-010's manual path working, and updates status with a Supersedes header
- ADR-012's roadmap discipline (7 independently-mergeable blocks, risk-ordered, with commit hashes recorded per item and a dead-code inventory in Appendix B) is genuinely good release engineering for a solo+AI project
- Feature growth is demand-driven, not speculative: KOI-0001 (UDP bridging, HTTP self-hosting) exists because a named consumer (zen-garden ORCH-0001/2/3) needed it, with the dependency declared in the proposal header
- The proposal→implementation→ADR→multi-agent-review→consolidation pipeline caught real defects (failing test, zero-coverage crate, 87 security findings) and closed the loop with verification audits

## Concerns

- ADR-012 Block 4 contains false history: it claims cross-domain imports never existed ('implemented correctly from the start... ADR-011's description was prospective'), but git shows koi-health depended on koi-certmesh/koi-dns/koi-mdns/koi-proxy at ADR-011's commit (588b616) and commit aa979d4 (2026-03-18) removed 8 cross-domain deps. The audit confabulated rather than checking git — a systemic risk for AI-maintained decision records (docs/adr/012-consolidation-roadmap.md:124-133)
- ADRs 001-010 carry fabricated/reconstructed dates (2025-01-15 through 2025-12-15) but were all written in one commit on 2026-02-15 (8a63730), in a repo whose history starts 2026-02-07. Future readers will misread the project's actual decision timeline; nothing marks them as retrospective
- ADR-011 status is still 'Proposed' (docs/adr/011-security-architecture-overhaul.md:3) despite being ~90% implemented — status drift undermines trust in the Status field across the corpus
- ADR-013's reversal accepts the exact security cost ADR-010 was written to avoid (root-equivalent Docker socket) with no mitigation beyond documentation (docs/adr/013-runtime-adapters.md:124), and its metadata table specifies K8s/systemd/Incus mappings for backends that are stubs returning BackendUnavailable (crates/koi-runtime/src/lib.rs:252-258)
- ADR-012 has a duplicated section '### 7.7' (lines 298 and 302) — an unreviewed editing artifact in an 'executed' ADR
- Proposal organization is inconsistent: AUTH-ADAPTERS.md, SSE-EVENT-IDS.md, KOI-0001, KOI-0002 are all marked 'Implemented' but live outside docs/proposals/implemented/, while DOCKER-ADAPTER.md (status 'Idea') was superseded by ADR-013 yet carries no superseded marker
- The FIDO2 arc is a cautionary tale of speed over verification: AUTH-ADAPTERS proposed it (2026-02-13), it shipped, then ADR-011 (2026-03-18) found the unlock slot cryptographically broken (credential_id is public, so the storage key is derivable by any traffic observer) — roughly five weeks from design to 'quarantined as broken', with the dead code still in tree behind a feature flag
- docs/plans/security-architecture-overhaul.md (646 lines) duplicates ADR-011 content and is now fully executed but not archived — two sources of truth for the same decision
- Last commit is 2026-05-10 (one month of silence as of 2026-06-11) — deferred items (FIDO2 redesign, SSE replay, dashboard phases) have no owner or timeline

## Maturity signals

- Beta-process signal: a full security overhaul (ADR-011, 87 findings) was specified, executed, and re-verified within ~10 days, with verifiable commit trail — most hobby projects never do this once
- Beta-code signal: panic-free production paths verified (1 residual unwrap in 15 crates); 796 #[test] annotations; ADR-012 cites 807+ passing tests at 2026-03-25
- Alpha signal: ADR-011 and ADR-012 both explicitly accept breaking serde/wire changes with 'no migration path — acceptable given the experimental stage' / 'acceptable for pre-1.0 project' — the project self-identifies as pre-stability
- Alpha signal: FIDO2 unlock shipped broken and is quarantined behind an off-by-default feature flag pending a redesign that depends on immature Rust ecosystem support (ADR-012 items 7.2/7.5)
- Prototype-tell: ADRs 001-010 were written retroactively in a single batch with invented dates, meaning the first year of 'decision history' is reconstructed narrative, not record
- Feasibility-test tell: total elapsed development is ~3 months (2026-02-07 to 2026-05-10, 176 commits) for 15 crates / ~70 endpoints / ~50 commands — breadth massively outpaces hardening time per domain; certmesh alone has 19 endpoints and a failover protocol that was redesigned once already (empty passphrase → X25519)
- Speculative-surface tell: RuntimeBackendKind declares Systemd/Incus/Kubernetes variants that error at runtime; command-surface's channel-binding and parity-validation machinery was designed, then superseded before use (COMMAND-MANIFEST phases 5-9)

## Shed or simplify

- command-surface crate: COMMAND-MANIFEST phases 5-6 were superseded by a simpler inline design (ApiEndpoint in CommandDef), phase 7 (clap cross-validation) deferred, phases 8-9 (Zen Garden port, crates.io publish) deferred indefinitely, and KoiScope is dead code per ADR-012 Appendix B. A standalone generic-trait crate with glyph rendering for exactly one consumer is over-abstraction — fold the used subset into the binary crate or aggressively trim
- FIDO2 unlock slot + ctap-hid-fido2 CLI path: declared cryptographically broken (ADR-011 Root Cause 3), gated off by default, redesign blocked on Rust ecosystem maturity. Delete the broken slot code rather than carrying quarantined crypto in tree; the AuthAdapter trait (ADR-009) makes re-adding it cheap later
- RuntimeBackendKind::Systemd/Incus/Kubernetes stub variants and ADR-013's metadata-mapping table for unimplemented backends: remove the variants or mark the table explicitly aspirational — currently the enum and docs promise five runtimes while one exists (crates/koi-runtime/src/lib.rs:252-258)
- docs/plans/security-architecture-overhaul.md (646 lines): fully executed; archive it and let ADR-011 be the single record, updating ADR-011's status from 'Proposed' to 'Implemented'
- ADR housekeeping batch: fix duplicate §7.7 in ADR-012 (lines 298/302); correct Block 4's false 'was prospective' note to cite aa979d4; add a 'retrospectively documented 2026-02-15' header to ADRs 001-010; add Superseded-by-ADR-013 header to docs/proposals/DOCKER-ADAPTER.md; move the four root-level 'Implemented' proposals into docs/proposals/implemented/
- SSE replay ring buffer (ADR-006 phase 2, ADR-012 item 7.4) and dashboard phases 7.2-7.4: both deferred twice already — either drop them from the active record into an icebox doc or commit to them; carrying them as perpetual 'deferred' items inflates perceived scope
- The deferred-but-promised mTLS/certmesh endpoint set deserves a consolidation pass: 19 certmesh endpoints (CLAUDE.md API reference) for a homelab CA is the widest single-domain surface in the project; ADR-011's dual-port design already concentrates the inter-node subset, suggesting the rest could shrink
